一、总则

1.1 为规范公司档案管理，确保档案安全、完整、保密，防止档案信息泄露、丢失或损毁，特制定本制度。

1.2 本制度适用于公司所有纸质档案和电子档案的管理，包括但不限于业务合同、交易材料、诉讼材料、发票、财务资料、税务资料等。

1.3 档案管理遵循"统一管理、分级负责、安全保密、高效利用"的原则。

二、管理机构与职责

2.1 档案管理委员会

由CIO牵头，各部门负责人组成，负责档案管理政策的制定和监督执行。

2.2 档案管理部门

设立专职档案管理部门，配备专职档案管理员，负责档案的日常管理工作。

2.3 各部门职责

  • 各部门指定兼职档案管理员，负责本部门档案的收集、整理和移交工作
  • 部门负责人对本部门档案的安全保密负领导责任

三、档案分类与分级

3.1 档案分类

档案类别 内容说明 保存期限
业务合同类 各类业务合同、协议等 永久或合同期满后10年
交易材料类 交易记录、凭证等 10年
诉讼材料类 诉讼文件、判决书等 永久
财务税务类 财务报表、发票、税务资料等 10年
人事行政类 人事档案、行政文件等 永久或长期

3.2 档案分级

密级 定义 管理要求
绝密 一旦泄露会对公司造成特别严重损害的信息 严格限制知悉范围，单独存放，特殊审批
机密 一旦泄露会对公司造成严重损害的信息 限制知悉范围，加密存储，严格审批
秘密 一旦泄露会对公司造成损害的信息 控制知悉范围，加密或物理保护
内部 仅限公司内部使用的信息 公司内部流通，禁止外传
公开 可对外公开的信息 按一般文件管理

四、档案安全管理措施

4.1 物理档案安全管理

  • 设立专用档案室，配备防火、防盗、防潮、防虫、防尘设施
  • 档案室实行双人双锁管理，进出登记制度
  • 绝密档案单独存放于保险柜，由专人管理
  • 档案借阅需填写《档案借阅审批表》，经审批后方可借阅
  • 档案不得带离公司，特殊情况需经公司总经理审批

4.2 电子档案安全管理

  • 建立电子档案管理系统，实行统一管理
  • 电子档案存储于公司内部服务器，禁止存储于个人设备
  • 机密级以上电子档案必须加密存储
  • 电子档案访问实行权限分级管理，最小权限原则
  • 所有电子档案操作记录日志，保存至少6个月
  • 电子档案传输必须通过公司批准的安全渠道

4.3 档案流转控制

  1. 档案流转必须登记，记录流转时间、内容、接收人等信息
  2. 纸质档案流转必须使用专用档案袋，密封并加盖骑缝章
  3. 电子档案流转必须通过加密通道，并记录流转日志
  4. 绝密档案流转必须由专人护送或使用专用加密通道
  5. 外部流转必须经公司总经理审批，签订保密协议

4.4 档案销毁管理

  • 建立档案销毁审批制度，编制销毁清册
  • 纸质档案销毁必须碎纸或焚毁，两人以上监销
  • 电子档案销毁必须彻底删除并覆盖存储介质
  • 绝密档案销毁需报公司总经理批准并记录销毁过程

5.2.2 终端安全管控方案

核心组件：

  1. 设备控制：
    • 禁用所有外部存储设备（通过组策略/GPO）
    • 例外审批使用加密U盘（如IronKey D300）
  2. 屏幕水印：
    # 动态水印生成算法示例
    def generate_watermark(user, time):
        return f"{user} | {time} | {random.randint(1000,9999)} | {socket.gethostname()}"

    实现要求：每秒刷新+半透明叠加+多角度可见

  3. 行为监控：
    • 键盘活动记录（仅针对敏感应用）
    • 截屏/录屏检测（Hook系统API调用）
    • 打印内容审计（集成打印服务器日志）

5.2.3 网络层防护

三合一防护体系：

  1. 传输加密：
    • 强制TLS 1.3+国密算法
    • 邮件系统部署S/MIME证书
  2. 内容过滤：
    • 部署Forcepoint Web Security
    • 敏感词过滤规则（正则表达式示例）：
      (客户名单|报价单|合同草案)[\s\S]{0,50}(发送|外传|转发|拷贝)
  3. 异常检测：
    • 部署Darktrace企业免疫系统
    • 检测指标包括：
      • 非工作时间大量下载
      • 压缩包外发行为
      • 云存储上传流量突增

七、责任追究

  • 违反本制度，视情节轻重给予警告、记过、降级、开除等处分
  • 造成档案损毁、丢失或泄密的，依法追究法律责任
  • 部门发生档案安全事故的，追究部门负责人责任
